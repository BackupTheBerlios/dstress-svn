^ About

DStress is a collection of bug tracking and language conformance test cases for the D programming language.
The unexpected output of a test case can have various reasons:
  1. a bug in the test case
  2. a failure of the build environment
  3. an inconsistent D-language specification or misunderstanding thereof
  4. a feature introduced in a later D-language specification
  5. a real compiler bug
  6. a bug in the operating system or the hardware
Currently the dmd and gdc compilers are being tested.

The symbols are the same as those used by the DejaGnu test framework.
Condition: Passed
  1. compiler return code zero
  2. no assertion was thrown
  3. return code of main is zero (applies only to the runtime tests)
Condition: Failed
  1. compiler return code unequal zero
  2. linker return code unequal zero
  3. assertion was throwm
  4. return code of main isn't zero (applies only to the runtime tests)
Condition: Error
  1. compiler segfaulted
  2. test case segfaulted
  3. unexpected compiler failure without source indication
